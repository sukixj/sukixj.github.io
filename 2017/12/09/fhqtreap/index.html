fhq Treap（可持久化Treap）

发表于 2017-12-09 | 阅读次数:

这玩意真的是短小精悍qwq

光掌握splay是不够的，虽然它的功能很强大，但是常数太大了，所以我们需要一种速度比较快的平衡树，听老师讲了一遍Treap只后，我决定选择和一般Treap复杂度差不多的fhq Treap（无旋Treap）。

核心操作：

merge(x,y)

返回一棵树的根，以x为根的Treap和以y为根的Treap合并为一棵新的Treap，保证x中的所有元素小于等于y中的所有元素。

key[x]表示x节点的随机值，和一般Treap一样

首先明确返回的根不是x就是y。
1) key[x] >= key[y] -> x作为根 -> x的左儿子不会发生变化(满足性质) -> 右儿子merge(ch[x][1],y)。
2) key[x] < key[y] -> y作为根 -> 左儿子merge(x,ch[y][0]) -> 右儿子 不变。

递归实现。

split(x,k)

返回两棵树的根，把以x为根的Treap前k小或者小于k的数拿出来组成一棵新的Treap，剩下的组成一棵Treap，一棵Treap -> 两棵Treap。

以下所提到的第一棵树第二棵树均为操作中split出的两棵树。

  • 前k小：
    1) k <= size[x][0]（全部都在左子树里） -> split(ch[x][0],k) -> 返回第一棵树和以x为根以第二棵树为左子树以ch[x][1]为右子树的树两棵树。
    2) k > size[x][0] -> split(ch[x][1],k-size[ch[x][0]]-1) -> 返回以x为根以ch[x][0]和第一棵树为左右子树的树和第二棵树两棵树。。
  • 小于k：
    1) k >=val[x] -> 左子树不变 -> split(ch[x][1],k) -> 返回以x为根以ch[x][0]和第一棵树为左右子树的树和第二棵树两棵树。。
    2) k < val[x] -> 右子树不变 -> split(ch[x][0],k) -> 返回第一棵树和以x为根以第二棵树为左子树以ch[x][1]为右子树的树两棵树。

基本操作

new_node(x)

新开一个权值为x的节点。

insert(x)

插入一个权值为x的节点

先将小于x的数split出来，然后再分别把第一棵树，x，第二棵树merge。

del(x)

删除一个权值为x的节点

先将小于等于x的数split出来，然后再把x从第一棵树中split出去也就是split小于x的树，丢掉x，把剩余的两棵树merge。注意别把x的子树也都一块丢掉了（先merge）。

find(x)

查询大小为x数的排名

把小于x的树split出来，然后输出第一棵树的size+1就好了（怕是有重复元素），最后再merge起来

findx(x)

查询排名为x的数

直接找k小，和splay一模一样

pre(x)

查询权值为x的数的前驱

先把小于x的数split出来，然后在第一棵树中找最大值，也就是第size[第一棵树]个值，最后merge起来

next(x)

查询权值为x的数的后继

先把小于等于x的数split出来，然后在第二棵树中找最小值，也就是第1个值，最后merge起来

最后整合一下普通平衡术

区间操作

先把前r+1个数split出来，再把第一棵树的前l个数split出来，这样第一棵树的第二棵树就是我们的操作区间了，最后把它们都merge起来。

注意建树的时候要建[0,n]，[1,n]就会直接gg
